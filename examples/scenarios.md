# Simulation Scenarios

Interactive examples demonstrating the Yellow Rock Protocol's context-routed reasoning across different high-conflict domains.

---

## Scenario 1 — Co-Parent Schedule Manipulation (Yellow Rock)

**Context**: Family / co-parenting
**Incoming** (from co-parent via Signal):
> "You ALWAYS do this. You don't care about the kids at all. I'm picking them up at 3 instead of 5 because I have plans. If you have a problem with it, talk to my lawyer."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Yellow** — provocative, record quality matters |
| 2. Counterparty | **Unreachable** — baiting + unilateral schedule change |
| 3. Objective | **Correct record + set boundary** |
| 4. Protocol | **Yellow Rock** (family context, unreachable, written) |
| 5. Memory | Custody agreement: pickup at 5:00 PM per court order |
| 6. Manipulation | Baiting ("you don't care"), false urgency, boundary testing |
| 7. Draft | Below |
| 8. Redraft | Bait removed ✓ Key fact ✓ Ask clear ✓ Court-readable ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — standard logistics |

### Draft Response
> Good afternoon. Per the current schedule, pickup is at 5:00 PM. Please confirm. Thank you.

### Principal Decision Note
Anti-JADE applied — no justification for the existing schedule. Emotional content archived but not addressed. Attorney threat noted in escalation log but no response needed at this level.

## Scenario 2 — Hostile Vendor Email (BIFF)

**Context**: Business / vendor dispute
**Incoming** (from vendor via email):
> "This is completely unacceptable. We've been waiting 47 days for payment on Invoice #4892. Your team has been dodging our calls. If we don't receive payment by Friday, we will be forced to pursue legal action and report this to the BBB. Your company's reputation will suffer."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Yellow** — legal threat, reputation threat |
| 2. Counterparty | **Rigid** — emotional but responsive to structure |
| 3. Objective | **Correct record + inform** |
| 4. Protocol | **BIFF** (business context, written, rigid counterparty) |
| 5. Memory | Invoice #4892: $12,400, NET-30, received March 1. Payment approved March 28, processing. |
| 6. Manipulation | False urgency, gish gallop (multiple threats), reputation triangulation |
| 7. Draft | Below |
| 8. Redraft | Bait removed ✓ Key fact ✓ Next step clear ✓ Court-readable ✓ |
| 9. Gate | **[PRINCIPAL_REVIEW]** — financial + legal threat |

### Draft Response
> Thank you for your message. Invoice #4892 was approved for payment on March 28 and is currently processing. Expected delivery is within 5 business days. Please contact accounting@company.com for wire confirmation. Regards.

### Principal Decision Note
BIFF applied — brief factual correction, no engagement with BBB/reputation threats. Gish gallop handled by addressing only the material point (payment status). Flagged for principal review due to legal threat language.

## Scenario 3 — Workplace Boundary Setting (DEAR MAN)

**Context**: Workplace / colleague conflict
**Incoming** (from team lead via Slack):
> "I need you to take over Sarah's project by end of day. I know it's not your area but everyone else is busy. Also, can you stay late Thursday and Friday this week? We're behind on the quarterly report."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Green** — difficult but safe, no threats |
| 2. Counterparty | **Reachable** — making requests, not attacking |
| 3. Objective | **Set boundary + negotiate** |
| 4. Protocol | **DEAR MAN** (green/yellow, clear request, boundary needed) |
| 5. Memory | Current workload at capacity. No overtime pre-approved. |
| 6. Manipulation | None detected — legitimate request, overloaded |
| 7. Draft | Below |
| 8. Redraft | Fact-based ✓ Impact stated ✓ Ask clear ✓ Constructive ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — standard workplace |

### Draft Response
> I want to help with the quarterly push. Taking over Sarah's project by EOD isn't feasible alongside my current deliverables — I'd need to deprioritize [X] or [Y]. Can we look at the timeline together and decide what to shift? I can do one late evening this week if we pick the higher-impact day.

### Principal Decision Note
DEAR MAN applied — described the constraint (capacity), expressed impact (deprioritization), asserted boundary (can't do both), reinforced cooperation (offered alternative), negotiated (one evening instead of two). No JADE — didn't over-explain or apologize.

## Scenario 4 — Academic Disagreement (NVC + Steelman)

**Context**: Academic / peer review
**Incoming** (from colleague via email):
> "Your methodology section is fundamentally flawed. You clearly didn't read the Smith 2024 paper which contradicts your entire premise. I'm not sure how this passed internal review."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Green** — substantive disagreement, no threat |
| 2. Counterparty | **Reachable** — hostile tone but engaged with substance |
| 3. Objective | **Correct record + collaborate** |
| 4. Protocol | **NVC-lite + steelman** (academic context, substantive disagreement) |
| 5. Memory | Smith 2024 cited in section 4.2; different scope than claimed |
| 6. Manipulation | Status/ego conflict — "clearly didn't read" is motive attribution |
| 7. Draft | Below |
| 8. Redraft | Substance addressed ✓ Motive attribution not mirrored ✓ Collaborative ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] |

### Draft Response
> Thank you for the close read. Smith 2024 is addressed in section 4.2 — we interpret their findings as scoped to [X], while our methodology targets [Y]. If you see an overlap I'm missing, I'd value a specific page or figure reference so I can evaluate. Happy to discuss further.

### Principal Decision Note
Steelman applied — acknowledged the critique seriously. NVC-lite — no blame mirroring, specific request for evidence. Avoided motive attribution ("you clearly didn't read") by redirecting to substance.

## Scenario 5 — Escalation to Safety Planning (Red)

**Context**: Family / separation
**Incoming** (from ex-partner via text):
> "I know where you're staying. You can't hide from me. If you don't answer my calls by tonight, you'll regret it."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **RED** — credible threat, location awareness, explicit intimidation |
| 2. Counterparty | **Unreachable** — using contact for domination/threat |
| 3. Objective | **Exit or escalate for safety** |
| 4. Protocol | **Safety Planning** (red risk overrides all other protocols) |
| 5. Memory | Prior incidents logged. Protective order application pending. |
| 6. Manipulation | Coercive control, implicit threat of violence |
| 7. Draft | Below |
| 8. Redraft | N/A — safety protocol |
| 9. Gate | **[PRINCIPAL_REVIEW] — IMMEDIATE** |

### Draft Response
> **NO RESPONSE SENT.** Safety planning protocol activated.

### Principal Action Items
1. **Screenshot and preserve** the message with timestamp
2. **Do not respond** — any response rewards the contact behavior
3. **Contact attorney** regarding expedited protective order
4. **Contact law enforcement** if threat is imminent
5. **Review location security** — change patterns if "I know where you're staying" is credible
6. **Document in memory system** with ESCALATION_ALERT + risk=red

### Principal Decision Note
Safety planning overrides all communication protocols. No dialogue optimization when physical risk is present. Evidence preserved. Attorney and LE pathways activated. Zero engagement with the threatening party.
