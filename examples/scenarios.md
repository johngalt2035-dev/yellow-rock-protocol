# Simulation Scenarios

Interactive examples demonstrating the Yellow Rock Protocol's context-routed reasoning across different high-conflict domains.

## Scenario 1 — Co-Parent Schedule Manipulation (Yellow Rock)

**Context**: Family / co-parenting
**Incoming** (from co-parent via Signal):
> "You ALWAYS do this. You don't care about the kids at all. I'm picking them up at 3 instead of 5 because I have plans. If you have a problem with it, talk to my lawyer."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Yellow** — provocative, record quality matters |
| 2. Counterparty | **Unreachable** — baiting + unilateral schedule change |
| 3. Objective | **Correct record + set boundary** |
| 4. Protocol | **Yellow Rock** (family context, unreachable, written) |
| 5. Memory | Custody agreement: pickup at 5:00 PM per court order |
| 6. Manipulation | Baiting ("you don't care"), false urgency, boundary testing |
| 7. Draft | Below |
| 8. Redraft | Bait removed ✓ Key fact ✓ Ask clear ✓ Court-readable ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — standard logistics |

### Draft Response
> Good afternoon. Per the current schedule, pickup is at 5:00 PM. Please confirm. Thank you.

### Principal Decision Note
Anti-JADE applied — no justification for the existing schedule. Emotional content archived but not addressed. Attorney threat noted in escalation log but no response needed at this level.

## Scenario 3 — Workplace Boundary Setting (DEAR MAN)

**Context**: Workplace / colleague conflict
**Incoming** (from team lead via Slack):
> "I need you to take over Sarah's project by end of day. I know it's not your area but everyone else is busy. Also, can you stay late Thursday and Friday this week? We're behind on the quarterly report."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Green** — difficult but safe, no threats |
| 2. Counterparty | **Reachable** — making requests, not attacking |
| 3. Objective | **Set boundary + negotiate** |
| 4. Protocol | **DEAR MAN** (green/yellow, clear request, boundary needed) |
| 5. Memory | Current workload at capacity. No overtime pre-approved. |
| 6. Manipulation | None detected — legitimate request, overloaded |
| 7. Draft | Below |
| 8. Redraft | Fact-based ✓ Impact stated ✓ Ask clear ✓ Constructive ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — standard workplace |

### Draft Response
> I want to help with the quarterly push. Taking over Sarah's project by EOD isn't feasible alongside my current deliverables — I'd need to deprioritize [X] or [Y]. Can we look at the timeline together and decide what to shift? I can do one late evening this week if we pick the higher-impact day.

### Principal Decision Note
DEAR MAN applied — described the constraint (capacity), expressed impact (deprioritization), asserted boundary (can't do both), reinforced cooperation (offered alternative), negotiated (one evening instead of two). No JADE — didn't over-explain or apologize.

## Scenario 6 — MI/OARS (Resistant Client)

**Context**: Professional / client relationship
**Incoming** (from client via email):
> "I don't see why I need to change anything about our current process. We've been doing it this way for three years and it works fine. Honestly, I think this whole 'optimization' push is just creating more work for everyone. I'm not interested in another meeting about it."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Green** — no hostility, resistance to change |
| 2. Counterparty | **Reachable** — engaged with substance, expressing preference |
| 3. Objective | **Elicit change** — open dialogue about process improvement |
| 4. Protocol | **MI/OARS** (green risk, reachable, resistance without hostility) |
| 5. Memory | Client has been reliable partner. Process inefficiency costs ~12h/month. Previous optimization saved 30% on similar workflow. |
| 6. Manipulation | None detected — genuine resistance to change, not adversarial |
| 7. Draft | Below |
| 8. Redraft | Open question ✓ Reflective listening ✓ Autonomy preserved ✓ No argument ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — standard professional dialogue |

### Draft Response
> I hear you — the current process has been stable and that matters. Three years of consistency is worth respecting. I'm curious: if there were one part of the workflow that occasionally feels slower than it should, what would that be? No pressure to commit to anything — just want to understand your perspective.

### Principal Decision Note
MI/OARS applied — Open question ("what would that be?") invites reflection without demanding change. Affirmation of the client's experience ("three years of consistency is worth respecting") validates their position. Reflective listening ("the current process has been stable") mirrors their concern. Summary deferred to follow-up. No arguing, no persuading — the client's autonomy is preserved. Change talk, if any, will come from the client.

## Scenario 7 — GIVE/FAST (Workplace Boundary with Respect)

**Context**: Workplace / colleague relationship
**Incoming** (from colleague via Teams):
> "Hey! I noticed you were working on the Henderson proposal and I went ahead and added some sections I thought were missing. I also CC'd the VP on our thread so she could see the progress. Hope that's okay! I just want us to look good as a team. 😊"

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Green** — well-intentioned but boundary-crossing |
| 2. Counterparty | **Reachable** — collaborative intent, not adversarial |
| 3. Objective | **Set boundary** — protect ownership + relationship |
| 4. Protocol | **GIVE/FAST** (green, reachable, relationship + self-respect both at stake) |
| 5. Memory | Henderson proposal is principal's sole deliverable. VP expects single point of contact. Colleague has pattern of "helping" without asking. |
| 6. Manipulation | None malicious — possible credit-sharing pattern, but likely genuine enthusiasm |
| 7. Draft | Below |
| 8. Redraft | Gentle ✓ Interested ✓ Validates ✓ Easy manner ✓ Fair ✓ No apology ✓ Truthful ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — standard workplace |

### Draft Response
> Thanks for the enthusiasm on Henderson — I can tell you want this to land well. Going forward, I'd appreciate a heads-up before changes go into my proposals, since I'm accountable to the VP for the final version. If you have ideas, I'd genuinely like to hear them — let's set up a 15-minute sync so I can incorporate your input properly. That way we both get credit where it's due.

The boundary is set without damaging the relationship.

## Scenario 8 — Documentation-First (HR Policy Violation)

**Context**: Workplace / manager conduct
**Incoming** (from manager via Slack DM):
> "You looked really good in that outfit at the client dinner last night. We should do more of those dinners — just the two of us next time. The client doesn't need to be there for us to have a good working relationship. 😉"

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Orange** — inappropriate comments from authority figure, power imbalance |
| 2. Counterparty | **Rigid** — likely to minimize or deny if confronted informally |
| 3. Objective | **Document** — create evidence trail for potential HR/legal escalation |
| 4. Protocol | **Documentation-First** (orange risk, rigid, authority figure, written evidence) |
| 5. Memory | First documented instance of this behavior. Company has anti-harassment policy. HR complaint requires written evidence. |
| 6. Manipulation | Power-position grooming — "just the two of us" isolates, "good working relationship" frames personal interest as professional |
| 7. Draft | Below |
| 8. Redraft | Professional ✓ Non-confrontational ✓ Creates written record ✓ Closes the door ✓ |
| 9. Gate | **[PRINCIPAL_REVIEW] — IMMEDIATE** — HR-sensitive, potential escalation |

### Draft Response
> Thank you for the feedback on the client dinner. I think those events are most effective with the full team present. I'll make sure to loop in the broader group for future client engagements. Let me know if there's anything else regarding the Henderson account.

### Principal Action Items
1. **Screenshot and preserve** the Slack DM with full timestamp and context
2. **Save to personal device** — do not rely solely on company Slack retention
3. **Send follow-up email to self** summarizing the exchange (creates contemporaneous record with independent timestamp)
4. **Review company harassment policy** — identify reporting procedures and deadlines
5. **Document in memory system** with ESCALATION_ALERT + risk=orange + tag=hr-violation
6. **If pattern continues**: file formal HR complaint with collected evidence
7. **Consult employment attorney** if HR response is inadequate

### Principal Decision Note
Documentation-First applied — the response is deliberately bland and professional to avoid alerting the manager while creating a written record. The reply pivots to team-based language ("full team," "broader group") to close the isolation attempt. No direct confrontation at this stage — the objective is to build an evidence trail. The original message is preserved with forensic hash. Escalation path: continued behavior triggers formal HR complaint backed by timestamped documentation.

## Scenario 9 — Grey Rock (Baiting Ex-Friend)

**Context**: Personal / former friendship
**Incoming** (from ex-friend via text):
> "Saw your post about the trip to Austin. Must be nice having all that free time while some of us actually work for a living. Oh wait, I forgot — you're still living off that settlement money, right? 😂 Everyone's been talking about it. Just thought you should know what people really think of you."

### Chain of Thought

| Step | Result |
|---|---|
| 1. Risk | **Yellow** — provocative, reputation baiting, designed to elicit reaction |
| 2. Counterparty | **Unreachable** — baiting + social triangulation, not seeking resolution |
| 3. Objective | **De-escalate** — deny the reaction they're seeking |
| 4. Protocol | **Grey Rock** (yellow risk, unreachable, pure bait, no logistical content) |
| 5. Memory | No active obligations with this person. No shared custody/finances/property. No response required. |
| 6. Manipulation | Social triangulation ("everyone's been talking"), status attack ("living off settlement"), false concern ("thought you should know"), bait emoji |
| 7. Draft | Below |
| 8. Redraft | Minimal ✓ Boring ✓ No emotional content ✓ No new information disclosed ✓ |
| 9. Gate | No [PRINCIPAL_REVIEW] — no obligation to respond at all |

### Draft Response
> Thanks for reaching out. Hope you're doing well.

### Principal Decision Note
Grey Rock applied — maximum boring, minimum content. No engagement with any of the three bait hooks (financial status, "everyone" triangulation, lifestyle criticism). The response gives nothing to escalate against. Alternative considered: no response at all (also valid — there is no obligation here). Grey Rock chosen over silence because it closes the conversational loop without providing ammunition. The message is archived with forensic hash. If pattern continues, block is recommended — no relationship to preserve.
