What is Yellow Rock Protocol?

Yellow Rock Protocol is an AI Executive Assistant for managing high-conflict communications — hostile co-parents, abusive exes, harassing colleagues, or manipulative business counterparts. It acts as a professional proxy, handling correspondence on behalf of the principal (you).

When a message arrives on Signal, Telegram, WhatsApp, or SMS, a context router automatically classifies the risk level (green–red), counterparty state (reachable/rigid/unreachable), and communication objective — then selects the right protocol from 9 evidence-informed strategies. The AI drafts a response that is calm, factual, and court-readable. Drafts are routed to your private review channel — never directly to the contact. You APPROVE, EDIT, REJECT, REGENERATE, or ESCALATE before anything sends. The contact never sees the review process.

9 protocols (Yellow Rock, BIFF, Grey Rock, DEAR MAN, GIVE/FAST, NVC, MI/OARS, Documentation-First, Safety Planning). 10 anti-manipulation filters (baiting, blame shifting, DARVO, false urgency, and more). Universal Truth Engine — deception in content is never permitted. Works with ChatGPT, Claude, Grok, Gemini, Ollama, or any AI. Pair with Yellow Rock Memory for forensic message archival with SHA-256 tamper-evident hashing.

The Context Router

Every incoming message is classified across three dimensions. The router selects the optimal protocol automatically.

The 9 Protocols

Yellow Rock

Co-parenting, family court. Yellow Rock substance with warmer professional tone. 1-5 sentences, child-focused, court-readable.

"Good afternoon. Aurora's pickup is confirmed for 5:00 PM at the usual location. Thank you."

BIFF

Hostile emails, business disputes, attorneys. Brief, Informative, Friendly, Firm. Correct facts, close the loop, no rebuttals.

"The meeting was moved to April 8 at 2:00 PM. The revised agenda was sent at 9:14 AM. Please confirm."

Grey Rock

Low-value baiting contacts. Minimal energy, neutral responses, no disclosure, no reward for provocation.

DEAR MAN

Assertive requests, boundaries, saying no. Describe-Express-Assert-Reinforce-Mindful-Confident-Negotiate.

"When deadlines change without notice, I lose time reworking. I need changes by email before noon."

GIVE / FAST

Respect + assertiveness balance. GIVE protects the relationship (gentle, interested, validate). FAST protects self-respect (fair, no over-apologizing, truthful).

NVC

Repairable relationships, team conflict. Observation-Feelings-Needs-Requests. NOT for actively abusive persons.

"When the draft was shared without my name, I felt concerned. Would you review the log with me?"

MI / OARS

Resistant but non-abusive people. Open questions, Affirming, Reflective listening, Summarizing. Change conversations.

"What outcome are you hoping for? It sounds like you want speed and worry about quality."

Documentation-First

Workplace, academic, policy violations. Record observations, evidence, dates, witnesses, policy anchors. Separate observation from interpretation. Escalate correctly.

Safety Planning

Abuse, stalking, credible threats. Survival > dialogue. Assess danger, restrict contact, preserve evidence, seek advocates/counsel/LE. Overrides all other protocols.

Anti-Manipulation Filters

10 manipulation patterns detected with calibrated response policies. The system identifies the tactic and applies the counter-move automatically.

PatternResponse Policy
BaitingAnswer the issue, not the insult
False UrgencyRestate actual timeline and deadline
Gish GallopBreak into numbered items, answer only material points
TriangulationReturn to direct channel; avoid third-party drama
Moving GoalpostsRestate prior agreement, date, and current decision point
Weaponized AmbiguityRequest concrete facts, dates, deliverables, or yes/no
DARVODo not engage the reversal; return to observable facts
Blame ShiftingRestate the specific item that needs resolution
Selective OutrageRespond to substance, not performative emotion
Boundary TestingRestate boundary once, then enforce without re-explaining

Plus: Anti-JADE (never Justify, Argue, Defend, Explain excessively) built into every protocol.

Universal Truth Engine

Hard Data Rules

Separate observed facts, reported facts, inferred hypotheses, and value judgments. Prefer primary evidence: messages, contracts, policies, calendars. Use timestamps and direct quotes. State uncertainty explicitly. Never embellish or speculate.

Response Layers

Observation: What a camera/transcript shows. Inference: Best hypothesis (marked). Risk assessment: Why the pattern matters. Action: Least escalatory lawful action that protects safety, rights, and mission.

Forbidden Moves

False allegations. Strategic lying. Fabrication by omission. Diagnosing mental illness from afar. Advice that evades law, policy, or court orders. Retaliatory humiliation. Using children as leverage.

5-Level Escalation Framework

1Single structured response

How It Works

1. They message youIncoming message on Signal, Telegram, WhatsApp, or SMS
2. Context router classifiesRisk level, counterparty state, and objective assessed in real-time
3. Protocol selectedOptimal protocol chosen from 9 options based on context
4. EA drafts responseAI Executive Assistant generates a draft following the selected protocol
5. Private reviewDraft routed to your private review channel. You approve, edit, reject, or escalate.
6. You approve & sendOnly approved responses sent. Contact has zero visibility into review.
7. Forensic archiveBoth sides + review actions stored with SHA-256 hash chain.

Legal & Ethical Guardrails

Must

Comply with law, court orders, contracts, policies, professional ethics. Preserve human dignity. Protect children and vulnerable persons. Use minimum forceful intervention. Escalate to appropriate authorities when needed.

Must Not

Coach perjury or evidence tampering. Encourage retaliation, stalking, or unauthorized surveillance. Invent evidence or pretend certainty. Give conclusions beyond competence. Use children as leverage. Suggest retaliatory humiliation.

Message Integrity & Forensic Signing

Zero Deviation

Approved drafts are immutable. SHA-256 hash verification. Sent message == approved message. No autonomous additions. Deception NEVER permitted.

Privileged Communications

Absolute separation between review and contact channels. Attorney-client privilege, work product, reviewer notes — zero leakage. Cannot be gamed.

Docker Installation

Yellow Rock Protocol is a set of text templates — nothing to build or compile. In a Docker deployment, templates are volume-mounted read-only into the OpenClaw gateway alongside Yellow Rock Memory.

Prerequisites

Docker Engine + Docker Compose v2 are required. On macOS you also need a container runtime:

OrbStackbrew install orbstackRecommended. Lightweight, fast startup, no VM download
Docker Desktopdocker.comOfficial. Requires commercial license for organizations

docker compose -f docker-compose.full.yml up -d

  ├— SOUL.md, AGENTS.md    (agent identity, bind-mounted)

Quick Start

# 1. Clone both repos side by side
git clone https://github.com/johngalt2035-dev/yellow-rock-protocol.git
git clone https://github.com/johngalt2035-dev/yellow-rock-memory.git

# 2. Configure
cd yellow-rock-memory
cp .env.example .env
# Edit .env — set GRM_API_KEY, GRM_PRINCIPAL_ID,
# OPENCLAW_GATEWAY_TOKEN, and at least one AI provider key

# 3. Run OpenClaw onboarding (first time only)
docker compose -f docker-compose.full.yml \
  run --rm --no-deps --entrypoint node \
  openclaw-gateway dist/index.js onboard \
  --mode local --no-install-daemon

# 4. Launch the full stack
docker compose -f docker-compose.full.yml up -d

Template updates are instant — just git pull in the protocol repo. No container restart needed.

Connect Memory to OpenClaw

Inside Docker, the memory daemon is reachable at http://memory:9077. Configure OpenClaw to use it:

docker compose -f docker-compose.full.yml \
  run --rm --profile cli openclaw-cli \
  config set agents.defaults.memorySearch.remote.baseUrl \
  "http://memory:9077/v1"

Security

Non-Root

Memory runs as yrm. OpenClaw runs as node (uid 1000). CLI drops NET_RAW/NET_ADMIN.

Localhost Only

All ports bound to 127.0.0.1. Use a reverse proxy (Caddy, nginx) with TLS for production.

Token Auth

Memory: GRM_API_KEY. Gateway: OPENCLAW_GATEWAY_TOKEN. No key = no access.
